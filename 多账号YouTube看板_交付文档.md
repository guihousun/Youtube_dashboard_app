# 多账号 YouTube 公开数据看板交付文档

## 1. 项目定位

本项目是一个 **纯公开数据版 YouTube 多频道管理看板**。

适用对象：

- 老板 / 管理者
- 运营负责人
- 需要横向对比多个员工频道的人

核心特点：

- 不登录 YouTube 账号
- 不需要 OAuth
- 只抓公开频道与公开视频数据
- 支持频道别名
- 支持多频道统一看板

## 2. 当前保留的功能

### 01 / 频道总播放量较昨日快照增量排行

说明：

- 基于本地每日快照计算
- 没有昨日快照的频道不参与排行
- 会显示 `新接入 / 无昨日基准`

### 02 / 频道当前订阅数排行

说明：

- 使用频道公开订阅数

### 03 / 昨日发布视频当前总播放排行

说明：

- 昨天发布
- 今天查看它的当前总播放

### 04 / 今日发布视频汇总

展示：

- 今日发布视频数
- 今日视频总播放
- 今日视频总点赞
- 今日视频总评论

### 05 / 昨日发布视频封面墙

说明：

- 显示全部昨日发布视频
- 每张卡显示当前播放量
- 点击可打开 YouTube

### 10 / 频道公开画像总览

展示：

- 频道名
- 别名
- handle
- 头像
- 创建时间
- 地区
- 总播放
- 订阅数
- 视频数
- 昨日播放差值

### 11 / 频道管理

支持：

- 添加公开频道
- 删除频道
- 设置别名

## 3. 代码保留但默认不展示的功能

以下功能代码仍保留，但老板页面默认不显示：

- 频道当前视频总数排行
- 近 28 天发布视频总播放榜
- 近 28 天发布视频点赞榜
- 近 28 天发布视频评论榜

同时，后端默认也不自动生成这些扩展榜单，目的是节省额度。

如果后续要恢复，可通过后端参数重新开启。

## 4. 技术方案

- 后端：`Python`
- 前端：`HTML/CSS/JavaScript`
- 数据来源：`YouTube Data API v3`
- 凭证：`YT_API_KEY`

## 5. 目录说明

- `youtube_dashboard_app.py`：后端服务
- `dashboard-prototype.html`：前端页面
- `setup_project.py`：初始化脚本
- `.env.example`：API key 模板
- `accounts.json`：频道配置
- `snapshots/`：本地每日快照

## 6. 配置方式

只需要在 `.env` 中填写：

```env
YT_API_KEY=your_youtube_data_api_key
```

不需要：

- `Client ID`
- `Client Secret`
- 登录授权

## 7. 启动方式

推荐方式：

1. 双击 `一键配置环境.bat`
2. 输入 `YT_API_KEY`
3. 双击 `启动看板.bat`

## 8. 关于昨日快照

`01 / 频道总播放量较昨日快照增量排行`

依赖：

- `snapshots/昨天日期.json`

计算方式：

```text
今天 totalViews - 昨日快照 totalViews
```

说明：

- 新频道第一天不参与排行
- 没有昨日基准时显示 `新接入 / 无昨日基准`
- 负数不显示，统一按 0 处理
