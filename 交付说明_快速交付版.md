# YouTube 公开数据看板交付说明

## 1. 交付方式

请直接交付整个项目文件夹。

当前版本已经支持：

- 相对路径运行
- 任意目录解压后使用
- 一键初始化
- 一键启动

## 2. 当前版本是什么

这是 **纯公开数据版**。

含义：

- 只抓取公开频道与公开视频信息
- 不登录账号
- 不做 OAuth 授权
- 不读取私有 Analytics 数据

## 3. 对方拿到项目后怎么用

1. 双击 `一键配置环境.bat`
2. 输入 `YT_API_KEY`
3. 双击 `启动看板.bat`
4. 浏览器打开页面
5. 在页面底部添加频道：
   - `@handle`
   - 可选别名

## 4. 交付时应该包含的主体文件

建议上传这些：

- `youtube_dashboard_app.py`
- `dashboard-prototype.html`
- `setup_project.py`
- `.env.example`
- `accounts.json`
- `启动看板.bat`
- `一键配置环境.bat`
- `README_快速开始.md`
- `交付说明_快速交付版.md`
- `新电脑从零开始使用指南.md`

## 5. 不建议上传的内容

不要上传：

- `.env`
- `tokens/`
- `youtube_tokens.json`
- `snapshots/`
- `output/`
- `release/`
- `YouTubeDashboard_Portable.zip`

这些内容要么包含私密信息，要么是本地运行产物。

## 6. 用户需要准备什么

只需要：

- Python
- Node.js / npx
- 一个 `YouTube Data API v3` 的 `API Key`

## 7. 当前保留的看板内容

- 频道总播放量较昨日快照增量排行
- 频道当前订阅数排行
- 昨日发布视频当前总播放排行
- 今日发布视频汇总
- 昨日发布视频封面墙
- 频道公开画像总览
- 频道管理

## 8. 当前不显示但代码保留的能力

以下能力代码仍保留，但默认不展示：

- 频道当前视频总数排行
- 近 28 天发布视频总播放榜
- 近 28 天发布视频点赞榜
- 近 28 天发布视频评论榜

## 9. 关于“昨日快照增量”

这个字段不是 YouTube 官方提供的，而是本地通过快照计算：

```text
今天总播放量 - 昨日快照总播放量
```

所以：

- 没有昨日快照的频道不参与 01 排行
- 页面会显示 `新接入 / 无昨日基准`
- 建议每天至少运行一次

## 10. 当前交付结论

当前项目已经适合作为：

- 多频道公开数据监控看板
- 老板视角的频道管理看板
- 不登录、不授权的公开数据方案
